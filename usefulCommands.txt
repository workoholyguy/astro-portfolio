### Step 1: Set Up Your GitHub Repository

1. **Create a new GitHub repository** or navigate to your existing one.
2. **Clone the repository locally**:

   ```sh
   git clone https://github.com/<your-username>/<your-repository>.git
   cd <your-repository>
   ```

2. **Generate an SSH key pair**:

   ```sh
   ssh-keygen -t rsa -b 4096 -C "your-email@example.com"
   ```

3. **Add the public key to the `authorized_keys` file**:

   ```sh
   cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
   ```

4. **Copy the contents of your public key** (`~/.ssh/id_rsa.pub`).

### Step 3: Add SSH Key to GitHub

1. **Go to your GitHub repository**.
2. **Navigate to "Settings" > "Deploy keys"**.
3. **Click "Add deploy key"**.
4. **Title the key appropriately** and paste the contents of your public key.
5. **Check "Allow write access"** and click "Add key".

### Step 4: Add SSH Key and Other Secrets to GitHub Actions

1. **Go to your GitHub repository**.
2. **Navigate to "Settings" > "Secrets and variables" > "Actions"**.
3. **Add the following secrets**:
   - `IONOS_HOST`: Your IONOS host (e.g., `access1006262289.webspace-data.io`)
   - `IONOS_USERNAME`: Your IONOS username (e.g., `u116436539`)
   - `IONOS_SSH_PRIVATE_KEY`: The contents of your private SSH key (`~/.ssh/id_rsa` on your IONOS server)
   - `GH_PAT`: Your GitHub Personal Access Token with repository permissions

### Step 5: Create GitHub Actions Workflow

1. **Create a `.github/workflows/deploy.yml` file** in your repository:

   ```yaml
   name: Deploy Website

   on:
     push:
       branches:
         - main  # Deploy when changes are pushed to the main branch

   jobs:
     deploy:
       runs-on: ubuntu-latest

       steps:
         - name: Checkout code
           uses: actions/checkout@v2
           with:
             token: ${{ secrets.GH_PAT }}

         - name: Execute SSH command to deploy
           uses: appleboy/ssh-action@master
           with:
             host: ${{ secrets.IONOS_HOST }}
             username: ${{ secrets.IONOS_USERNAME }}
             key: ${{ secrets.IONOS_SSH_PRIVATE_KEY }}
             port: 22
             script: |
               cd /kunden/homepages/44/d1006262289/htdocs/test
               ./deploy.sh
   ```

### Step 6: Create Deployment Script on IONOS Server

1. **SSH into your IONOS server**:

   ```sh
   ssh u<IONOS-username>@<IONOS-host>
   ```

2. **Navigate to your project directory** and create a `deploy.sh` script:

   ```sh
   cd /kunden/homepages/44/d1006262289/htdocs/test
   nano deploy.sh
   ```

3. **Add the following content to `deploy.sh`**:

   ```sh
   #!/bin/bash
   # Navigate to your web directory
   cd /kunden/homepages/44/d1006262289/htdocs/test

   # Start the SSH agent and add the SSH key
   eval "$(ssh-agent -s)"
   ssh-add /kunden/homepages/44/d1006262289/htdocs/.ssh/id_rsa

   # Pull the latest changes from the main branch
   git pull origin main
   ```

4. **Make the script executable**:

   ```sh
   chmod +x deploy.sh
   ```

### Step 7: Commit and Push Changes to GitHub

1. **Commit and push the changes to your GitHub repository**:

   ```sh
   git add -A
   git commit -m "Setup continuous deployment"
   git push origin main
   ```

### Step 8: Verify the Deployment

1. **Go to your GitHub repository**.
2. **Navigate to the "Actions" tab**.
3. **Monitor the deployment workflow to ensure it completes successfully**.

### Step 9: Making Future Changes

2. **Commit and push the changes**:
